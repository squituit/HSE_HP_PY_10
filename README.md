## Краткое описание

Данная работа демонстрирует практическое исследование уязвимости **CVE-2021-41773** — критической ошибки нормализации пути в веб-сервере Apache HTTP Server версии 2.4.49, позволяющей обойти механизмы защиты и получить несанкционированный доступ к произвольным файлам файловой системы (Path Traversal).

Уязвимость была обнаружена в октябре 2021 года и затрагивает версии 2.4.49–2.4.50. Её особенность — обход фильтрации через **двойное URL-кодирование**, что делает её отличным учебным примером.

## Технический анализ уязвимости

### Суть проблемы

В Apache 2.4.49 изменена логика обработки закодированных последовательностей в пути запроса:

| Этап обработки         | Оригинальный путь                     | Результат после обработки |
|------------------------|---------------------------------------|---------------------------|
| Входящий запрос        | `/cgi-bin/.%2e/.%2e/etc/passwd`       |                           |
| Первое декодирование   | `/cgi-bin/.%2e/.%2e/etc/passwd`       | `%2e` → `.`               |
| Нормализация пути      | `/cgi-bin/../etc/passwd`              | `.%2e` → `..`             |
| Второе декодирование*  | `/cgi-bin/../../etc/passwd`           | **Обход фильтра!**        |

В уязвимой версии нормализация выполнялась **после** первого декодирования, но **до** второго, что позволяло обойти проверку `..` в пути.
